# Film Index — desktop (Tauri)

A thin [Tauri v2](https://v2.tauri.app) desktop wrapper around
**https://films.chrisj.uk**. It's a native window showing the site, plus two
commands the site calls when it detects it's running inside the app:

- **`open_in_player(url, title)`** — opens a film's stream URL in **mpv** if it's
  on `PATH`, otherwise the OS default player. The site's **Watch** button uses
  this (it reads "Watch in mpv" in the app).
- **`player_info()`** — returns `{ os, arch, mpv }`, shown on the film page.

Everything else (browsing, iGPU encode/stream, download, progress bars) is the
normal web app served remotely — this app just adds native playback. In a plain
browser the same **Watch** button plays in-browser instead, so the site works
with or without this app.

## Build it (on your viewing device — not the server)

Prereqs: **Rust** (https://rustup.rs) and the Tauri system deps for your OS
(see https://v2.tauri.app/start/prerequisites — e.g. `webkit2gtk` on Linux),
plus Node.

```sh
cd desktop
npm install
npm run tauri icon ../static/favicon.svg   # generate the app icons once
npm run dev                                 # run it
npm run build                               # produce an installable bundle
```

Log in through Cloudflare Access in the window the first time; the session
persists. On Linux you'll want `mpv` installed (`apt install mpv`) — otherwise it
falls back to whatever `xdg-open` is configured to use.

## Notes / things to verify

I could not build this on the (headless, Rust-less) server, so treat it as a
verified-by-reading starting point:

- **Loading a remote origin + calling commands** is configured via the window
  `url` (`tauri.conf.json`) and `remote.urls` in `capabilities/default.json`. If
  `invoke` is blocked, that capability block is the thing to check against the
  current Tauri docs (it's the "remote domain IPC" area).
- Icons are generated by `tauri icon` (above); the bundle step needs them.

## In-movie progress in mpv

Browser playback already saves your position. Tracking position during **mpv**
playback needs mpv's IPC socket (`--input-ipc-server`) polled from a command —
a deliberate follow-up, not wired here yet.
